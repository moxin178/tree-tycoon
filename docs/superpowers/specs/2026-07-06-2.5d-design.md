# tree-tycoon 2.5D 游戏世界设计文档

**日期**：2026-07-06  
**版本**：v1.0  
**状态**：待实现规划  

## 1. 核心概念

将现有扁平点击器改造为**俯视角 2.5D 自动化经营模拟游戏**。玩家扮演基地老板，在一片开放地图上建设木材产业链：伐木 → 加工木板 → 加工家具 → 装车出售。工人全自动运转，玩家主要负责买地、建建筑、雇工人、升级设施。

### 1.1 核心循环

```text
林区（工人伐木）→ 锯木厂（原木→木板）→ 家具厂（木板→家具）→ 仓库/门店 → 货车停车场（装满即走）
```

### 1.2 玩家职责

- 买地扩展可建造区域
- 建造林区、锯木厂、家具厂、仓库、停车场、道路
- 雇佣工人并分配到各个建筑
- 升级建筑、工人品质、货车容量等
- 控制老板角色在地图上巡视，提供临时生产增益

### 1.3 成功标准

- 玩家能清楚看到「谁在干什么」「资源在哪里」「钱从哪里来」
- 全自动工人不影响玩家离线或暂时离开
- 保留并扩展现有经济系统的深度

## 2. 架构与技术方案

### 2.1 渲染层

- 使用 **Canvas 2D** 渲染整个游戏世界
- HTML UI 层覆盖在 canvas 之上，用于面板和信息显示
- 所有角色、建筑、树木用矢量图形绘制，带阴影和简单动画

### 2.2 游戏循环

- `requestAnimationFrame` 驱动主循环
- 固定时间步长处理逻辑更新
- 每帧顺序：更新相机 → 更新实体 → 渲染世界 → 渲染 UI

### 2.3 核心模块

| 模块 | 职责 |
|------|------|
| `World` | 地图网格、地块状态、建筑位置、可行走区域 |
| `EntityManager` | 管理动态实体（工人、老板、货车、资源堆） |
| `WorkerAI` | 工人状态机与行为逻辑 |
| `Building` | 建筑基类及子类 |
| `Economy` | 金币、钻石、价格、成本、自动出售结算 |
| `Camera` | 跟随老板、拖拽平移、边界限制 |
| `InputManager` | 鼠标/触摸输入处理 |
| `SaveLoad` | 存档序列化与旧版迁移 |

### 2.4 状态管理

- 单例 `GameState` 保存运行时数据
- 复用现有经济字段，新增地图、建筑、工人等数据
- 保存时序列化到 localStorage

## 3. 游戏世界与地图

### 3.1 网格系统

- 每格大小：48×48 像素
- 初始地图：20×20 格，可扩展至 40×40 或更大
- 地块类型：空地、固定林区、建筑、道路、未购买地

### 3.2 林区设计

- 树林为**固定区域**，不是零散单棵树
- 伐木工人进入林区后自动伐木
- 树木被砍后**快速再生**，保持密度
- **无成熟度**：每棵树产出固定数量原木
- 可升级「林区等级」提高产出和再生速度

### 3.3 建筑与道路

- 所有建筑固定大小为 **2×3 格子**
- 建筑必须有一边邻接道路
- 道路必须连通到地图边缘预设的公路出入口
- 货车从出入口驶入，沿道路到达仓库停车场

### 3.4 买地机制

- 地图边缘有「未购买」地块，显示为灰色半透明
- 点击购买后解锁为空地
- 解锁价格随距离中心远近递增

### 3.5 建筑放置验证

1. 2×3 区域是否为空地
2. 至少一边邻接道路
3. 该道路是否连通到地图边缘

## 4. 建筑与加工链

### 4.1 建筑类型

| 建筑 | 大小 | 功能 | 输入 | 输出 |
|------|------|------|------|------|
| 林区 | 固定区域 | 工人伐木 | 无 | 原木 |
| 锯木厂 | 2×3 | 原木 → 木板 | 原木 | 木板 |
| 家具厂 | 2×3 | 木板 → 家具 | 木板 | 家具 |
| 仓库/门店 | 2×3 | 存储并出售货物 | 原木/木板/家具 | 金币 |
| 货车停车场 | 2×3 | 货车装货后离场 | 货物 | 金币 |
| 道路 | 1×1 | 连接建筑与出入口 | 无 | 无 |

### 4.2 加工链流程

```text
林区 ──原木──→ 锯木厂 ──木板──→ 家具厂 ──家具──→ 仓库/门店 ──货物──→ 货车停车场
                ↑_________也可以直接卖原木_________↑
                ↑_________也可以直接卖木板_________↑
```

### 4.3 货车出售机制

- 货车停在停车场，工人把货物搬上车
- 货车有容量上限（例如 50 单位）
- 装满后货车驶离地图，玩家获得金币
- 新货车几分钟后从出入口驶入
- 可升级「货车容量」和「货车频率」

### 4.4 建筑升级

- 所有建筑统一最高 **12 级**
- 每级提升：加工速度、库存容量、工人上限、外观变化

## 5. 工人 AI 与雇佣

### 5.1 雇佣规则

- **买断制**：一次性金币雇佣，无持续工资
- 每个建筑有**工人上限**，建筑升级可提高上限
- 可解雇工人

### 5.2 工人品质

- 等级：普通 → 优秀 → 史诗 → 传说
- 影响：移动速度、工作效率、搬运容量

### 5.3 升级渠道

| 渠道 | 消耗 | 限制 |
|------|------|------|
| 金币升级 | 金币 | 价格递增 |
| 钻石升级 | 钻石 | 钻石通过任务/广告/充值获得 |
| 广告升级 | 观看广告 | 每次有冷却时间，无次数上限 |

### 5.4 工人状态机

```text
Idle → FindTask → MoveTo → Work → Carry → DropOff → Idle
```

### 5.5 按工作场所的行为

#### 林区工人
1. 在林区内伐木
2. 搬运原木到最近的锯木厂或仓库
3. 放下原木后返回

#### 锯木厂工人
1. 从输入区取原木
2. 加工成木板
3. 搬运木板到最近的家具厂或仓库

#### 家具厂工人
1. 从输入区取木板
2. 加工成家具
3. 搬运家具到仓库

#### 仓库/停车场工人
1. 从仓库取货物
2. 搬运到停车场货车装车

### 5.6 寻路与碰撞

- 使用 A* 或简单网格路径查找
- 工人只在道路和空地上行走
- 不可穿过建筑
- 多个工人共享道路时简单处理碰撞

## 6. 玩家/老板角色与相机

### 6.1 老板角色

- 形象：戴安全帽的老板或伐木大亨
- 主要功能：代入感和临时增益
- 玩家点击空地，老板移动到该位置

### 6.2 老板参与生产

- 玩家可指挥老板去林区或建筑帮忙
- **不阻塞正常流程**：工人仍全自动运转
- 老板在场时提供临时效率加成（约 +20%）
- 增益持续 10-30 秒，或直到老板被派往新目标

### 6.3 相机行为

- 默认跟随老板移动
- 玩家可拖拽地图进入自由模式
- 停止拖动 3 秒后，相机缓慢回到老板位置
- 限制在地图边界内

### 6.4 输入优先级

1. 点击 UI 面板：不处理地图
2. 点击建筑：打开建筑面板
3. 点击老板：打开全局管理面板
4. 点击空地：老板移动到该位置

## 7. UI/UX 设计

### 7.1 顶部资源栏

固定显示：金币、钻石、原木、木板、家具、当前工人数量/上限

### 7.2 建筑面板

- **林区面板**：工人数/上限、雇佣、升级、工人列表
- **加工厂面板**：等级、升级、工人、输入/输出库存、效率
- **仓库面板**：库存、货车装载进度、下辆车到达时间、升级

### 7.3 建造菜单

- 点击空地弹出
- 可选：锯木厂、家具厂、仓库、停车场、道路
- 显示成本和解锁条件

### 7.4 老板管理面板

- 全局统计
- 设置
- 存档操作
- 商店（占位）

### 7.5 通知系统

- 右下角弹出消息
- 例如：货车装满、升级完成、建筑可升级

## 8. 存档与状态迁移

### 8.1 存档版本

- 将 `SAVE_VERSION` 从 3 升级到 4
- 新结构包含 world、workers、player、upgrades 等字段

### 8.2 旧存档处理

- 检测到旧版本时，提示玩家选择：
  - 继承部分资源（金币折算成启动资金）
  - 重新开始

### 8.3 自动保存

- 每 10 秒自动保存
- 重要操作后立刻保存
- 使用 `requestIdleCallback` 避免卡顿

### 8.4 容错

- 存档解析失败时自动生成新游戏
- 控制台输出警告

### 9.3 性能测试

- 模拟 50 个工人同时活动
